Architecture Overview

The Ethred backend follows a clean modular MVC architecture. Every API feature lives in its own folder under src/modules/ with three files: routes → controller → service. This keeps the codebase easy to extend without touching unrelated code.

HTTP Request
    │
    ├── Nginx (production) / localhost:5000 (dev)
    │
    ├── Express App (src/app.js)
    │   ├── Middleware: helmet, cors, cookie-parser, morgan, rate-limiter, passport
    │   └── Routes: /api/v1/*
    │
    ├── Modules (src/modules/)
    │   ├── auth/        routes → controller → service
    │   ├── users/       routes → controller → service
    │   ├── agencies/    routes → controller → service
    │   ├── properties/  routes → controller → service
    │   ├── favorites/   routes → controller
    │   ├── inquiries/   routes → controller
    │   ├── payments/    routes → controller
    │   └── admin/       routes → controller
    │
    ├── Config (src/config/)
    │   ├── db.js     → Prisma → PostgreSQL + PostGIS
    │   ├── redis.js  → ioredis → Redis
    │   ├── passport  → JWT + Google OAuth
    │   └── mailer    → Nodemailer SMTP
    │
    └── Socket.IO (src/sockets/) → Redis Pub/Sub adapter

Module Responsibility Map

Entry Point & App

FileResponsibility
server.jsBoots HTTP server, initializes Socket.IO, listens on PORT, handles graceful shutdown (SIGTERM/SIGINT) — closes DB, Redis, HTTP connections cleanly before exit.
src/app.jsBuilds the Express app: registers all middleware (CORS, Helmet, body parsers, Morgan, Passport, rate limiters), mounts all route groups, serves static uploads.

Config Layer (src/config/)

FileResponsibility
db.jsPrisma client singleton. Exports prisma for all modules. Exports connectDB() and disconnectDB() for lifecycle hooks.
redis.jsioredis client singleton. Exports setEx(key, ttl, val), get(key), del(key), and getRedisClient() for the Socket.IO adapter.
passport.jsConfigures two strategies: JWT Strategy (reads from HTTP-only cookie or Bearer header) and Google OAuth 2.0 (handles callback, creates user on first login).
mailer.jsCreates Nodemailer transporter via SMTP. Exports sendEmail({to, subject, html}) — used for OTPs, moderation alerts, inquiry notifications, and payment confirmations.

Middleware Layer (src/middleware/)

FileResponsibility
authenticate.jsValidates JWT (from cookie or Bearer header) via Passport. Attaches user to req.user. Rejects with 401 if missing or invalid.
authorize.jsFactory: authorize('ADMIN', 'SELLER'). Checks req.user.role against allowed list. Rejects with 403 if unauthorized.
errorHandler.jsCentralized error handler. Translates Prisma errors (unique → 409, not found → 404), Zod → 422, JWT → 401, custom ApiError, and fallback 500. All return consistent JSON.
upload.jsMulter-based uploader. Images → Sharp pipeline → WebP at 400/800/1200px widths. Documents (PDF) saved as-is. Exports uploadImages, uploadDocuments, uploadAvatar, uploadIdDocument.
rateLimiter.jsTwo limiters: generalLimiter (100 req/15 min) for all routes, strictLimiter (10 req/15 min) for auth routes. Prevents brute-force.

API Modules (src/modules/)

Each module has the same file pattern: routes.js → controller.js → service.js

🔐

auth/

Handles all authentication flows: email+password registration, OTP email verification, login, logout, password reset, and Google OAuth. Issues JWTs stored in HTTP-only cookies (15-min access, 7-day refresh).

POST /register POST /login POST /send-otp POST /verify-otp POST /forgot-password POST /reset-password GET /google GET /me
👤

users/

Manages user profiles, avatar uploads (transcoded to WebP), national ID / passport document uploads, and notification preferences. Supports both Ethiopian and international ID formats.

GET /profile PUT /profile POST /avatar POST /id-document GET /notification-prefs PUT /notification-prefs
🏢

agencies/

Full lifecycle for real estate agencies: creation with business license upload, employee invitation by email link, employee management, and aggregated analytics (listings, inquiry counts, conversion rates).

POST / GET /:id PUT /:id POST /:id/invite GET /:id/employees GET /:id/analytics
🏠

properties/

Core listing management. Public search with hierarchical filters (Region → City → Sub-City → Woreda). PostGIS geographic coordinates via raw SQL. Status state machine: DRAFT → PENDING → APPROVED/REJECTED. WebP image pipeline.

GET /search GET /:id POST / POST /:id/submit POST /:id/media/images GET /:id/stats
❤️

favorites/

Property wishlist management. Uses upsert for idempotent add operations. Returns paginated list with property thumbnails. Each user has an independent favorites list across all property types.

POST /:propertyId DELETE /:propertyId GET /
💬

inquiries/

Buyer ↔ Seller messaging system. Email notification to seller on new inquiry. Inquiry status workflow (NEW → IN_PROGRESS → RESOLVED). Scam report system — auto-suspends listing after 3+ independent reports.

POST / GET /received PATCH /:id/status POST /report/:propertyId
💳

payments/

Full Chapa ETB payment integration. Creates pending invoice → returns Chapa checkout URL → webhook receives payment confirmation → verifies HMAC-SHA256 signature → activates property promotion (featured 30 days).

POST /initiate GET /invoices GET /invoices/:id POST /chapa-webhook
🛡️

admin/

Full admin control panel. Property moderation queue (approve/reject/suspend) with seller email notifications. User management (ban, role change, identity verification). Agency approval. Full audit log viewer. Platform-wide dashboard stats.

GET /dashboard GET /properties/pending PATCH /properties/:id/approve GET /audit-logs PATCH /agencies/:id/approve

Real-time Layer (src/sockets/)

index.js — Initializes Socket.IO on the same HTTP port. Authenticates all clients via JWT on handshake. Uses a Redis Pub/Sub adapter for horizontal scaling. Manages rooms (inquiry:{id}), handles message:send, typing:start/stop, and personal notification rooms (user:{id}).

Utilities (src/utils/)

FileResponsibility
jwt.jssignToken(payload) and verifyToken(token) — wrappers around jsonwebtoken with configurable expiry.
otp.jsGenerates cryptographically random 6-digit OTP, stores in Redis with 5-minute TTL, validates and deletes on use.
password.jshashPassword(plain) and verifyPassword(plain, hash) — bcrypt at 12 rounds.
logger.jsWinston logger with console (dev) and rotating file transports for logs/combined.log and logs/error.log.
chapaClient.jsAxios wrapper for Chapa API. Exports initializePayment(), verifyTransaction(), and verifyWebhookSignature() (HMAC-SHA256).

Database (prisma/)

FileResponsibility
schema.prismaDefines all 11 database tables: users, profiles, agencies, agency_employees, properties, property_media, property_amenities, property_inquiries, user_favorites, billing_invoices, audit_logs. PostGIS geom_point column managed via raw SQL.
seed.jsCreates default Admin, Seller, and Buyer accounts. Creates a sample approved property in Bole, Addis Ababa with real PostGIS coordinates.

Running Locally Without Real Services

You can develop and test 100% of the backend without a real Gmail account, Google OAuth app, or Chapa credentials.

Step 1 — Start Docker (Database + Cache)

docker-compose up -d
docker ps    # Wait until both show (healthy)

This starts PostgreSQL 16 + PostGIS on port 5432 and Redis 7 on port 6379.

Step 2 — Apply Schema & Seed Data

npm run db:push     # Creates all 11 tables
npm run db:seed     # Creates admin, seller, buyer accounts + sample property

Step 3 — Configure .env for Local (No Real Credentials)

NODE_ENV=development
PORT=5000
FRONTEND_URL=http://localhost:3000
DATABASE_URL=postgresql://ethred_user:ethred_pass@localhost:5432/ethred_db?schema=public
REDIS_URL=redis://localhost:6379
JWT_SECRET=my_local_dev_secret_key_at_least_32_chars
JWT_EXPIRES_IN=15m
JWT_COOKIE_NAME=ethred_token

# Leave blank — OTP code will print to the console instead
SMTP_HOST=
SMTP_USER=
SMTP_PASS=

# Leave blank — Google login endpoint simply returns 404
GOOGLE_CLIENT_ID=
GOOGLE_CLIENT_SECRET=

# Leave blank — payment initiation returns an error gracefully
CHAPA_SECRET_KEY=
CHAPA_WEBHOOK_SECRET=

Step 4 — Start the Dev Server

npm run dev
# Server starts at http://localhost:5000
# Health check: http://localhost:5000/health

Testing Without Email (OTP Flow)

When SMTP_USER is blank, the OTP is printed directly to the server console instead of sent by email. Look for this in your terminal:

[info]: OTP for test@example.com: 847261

Testing Without Google OAuth

Use standard email/password instead. All other endpoints work identically:

# Register a new user
curl -X POST http://localhost:5000/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email":"dev@test.com","password":"Test@1234","full_name":"Dev User"}'

# Login (JWT cookie set automatically)
curl -c cookies.txt -X POST http://localhost:5000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@ethred.com","password":"Admin@1234"}'

# Call protected admin endpoint
curl -b cookies.txt http://localhost:5000/api/v1/admin/dashboard

# Public property search (no auth)
curl "http://localhost:5000/api/v1/properties/search?city=Addis+Ababa&category=APARTMENT"

Testing Payments (Chapa Sandbox)

Chapa provides a free sandbox environment for testing payments without real money:

  1. Register at chapa.co → Get a test key starting with CHASECK_TEST-...
  2. Set CHAPA_SECRET_KEY=CHASECK_TEST-... in .env
  3. Test card: 5399 8312 3400 0008, any future expiry, CVV 100
  4. For webhook testing locally, use ngrok: ngrok http 5000 → paste URL in Chapa dashboard

Visual Database Editor

npm run db:studio
# Opens Prisma Studio at http://localhost:5555
# Browse and edit all tables visually in your browser

Running the Test Suite

npm test               # Run all 6 tests
npm run test:coverage  # Run with HTML coverage report

Deploying to Render (Staging)

Render is ideal for a free staging environment to share with your team or testers.

⚠
PostGIS Limitation Render's free PostgreSQL tier does not include PostGIS. Use Neon (free, has PostGIS) or Render's paid PostgreSQL tier instead.

Step 1 — Push Code to GitHub

cd backend
git init
git add .
git commit -m "Initial Ethred backend"
git remote add origin https://github.com/YOUR_USERNAME/ethred-backend.git
git push -u origin main

Your .gitignore already excludes node_modules/, .env, uploads/, and logs/.

Step 2 — Create PostgreSQL with PostGIS on Neon

  1. Sign up at neon.tech (free tier)
  2. Create a new project → copy the Connection String
  3. In the Neon SQL editor, run: CREATE EXTENSION IF NOT EXISTS postgis;

Step 3 — Create Redis on Upstash

  1. Sign up at upstash.com (free tier)
  2. Create a Redis database → copy the REDIS_URL (starts with rediss://)

Step 4 — Create Web Service on Render

  1. Go to render.com → New → Web Service
  2. Connect your GitHub repository
  3. Set Build Command: npm install && npm run db:generate && npm run db:push && npm run db:seed
  4. Set Start Command: npm start

Step 5 — Set Environment Variables on Render

In the Render dashboard → your service → Environment tab, add all these variables:

NODE_ENV=production
PORT=10000
DATABASE_URL=postgresql://...   ← from Neon
REDIS_URL=rediss://...          ← from Upstash (rediss:// with SSL)
JWT_SECRET=<64-char random string>
JWT_EXPIRES_IN=15m
JWT_COOKIE_NAME=ethred_token
FRONTEND_URL=https://your-frontend.vercel.app
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=yourapp@gmail.com
SMTP_PASS=your_gmail_app_password
GOOGLE_CLIENT_ID=...
GOOGLE_CLIENT_SECRET=...
GOOGLE_CALLBACK_URL=https://ethred-api.onrender.com/api/v1/auth/google/callback
CHAPA_SECRET_KEY=CHASECK_TEST-...
CHAPA_WEBHOOK_SECRET=your_webhook_secret
UPLOAD_DIR=uploads

Step 6 — Deploy

Click Deploy. Render builds your app, pushes the schema, seeds the database, and starts your API.

Your API will be live at: https://ethred-api.onrender.com

ℹ
Free Tier Sleep Free Render web services spin down after 15 minutes of inactivity and take ~30 seconds to wake up. Use a paid plan or schedule a cron ping job (e.g., UptimeRobot) to keep it alive.

Server Setup

# SSH into your server
ssh root@YOUR_SERVER_IP

# Install Node.js 20
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt install -y nodejs

# Install PM2 globally
npm install -g pm2

# Install Docker
apt install -y docker.io docker-compose
systemctl enable docker

# Install Nginx
apt install -y nginx

# Install SSL tool
apt install -y certbot python3-certbot-nginx

Clone & Configure

mkdir -p /var/www/ethred
cd /var/www/ethred
git clone https://github.com/YOUR_USERNAME/ethred-backend.git .
npm install
cp .env.example .env
nano .env    # Fill in all production values

Start Database Containers

docker-compose up -d
npm run db:push      # Apply schema
npm run db:seed      # Seed initial data
npm run db:generate  # Generate Prisma client

Nginx Configuration

server {
    listen 80;
    server_name api.ethred.com;
    client_max_body_size 50M;

    location / {
        proxy_pass http://localhost:5000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # Serve uploads directly via Nginx (faster than Node)
    location /uploads/ {
        alias /var/www/ethred/uploads/;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
}
ln -s /etc/nginx/sites-available/ethred /etc/nginx/sites-enabled/
nginx -t && systemctl restart nginx

Enable HTTPS

# Point your domain DNS A record to server IP first, then:
certbot --nginx -d api.ethred.com
# Certbot auto-renews every 90 days

Start with PM2

cd /var/www/ethred
pm2 start server.js --name "ethred-api" --env production
pm2 save          # Save config to survive reboots
pm2 startup       # Register as system service (follow the output instruction)

PM2 Commands

pm2 status             # Check process status
pm2 logs ethred-api    # View real-time logs
pm2 restart ethred-api # Restart after code update
pm2 monit              # CPU/memory dashboard

Update Workflow

cd /var/www/ethred
git pull origin main           # Pull latest code
npm install                    # Install new packages
npm run db:generate            # Regenerate Prisma if schema changed
npm run db:push                # Apply schema changes
pm2 restart ethred-api         # Restart with zero downtime

Production Go-Live Checklist

  • NODE_ENV=production set in .env
  • JWT_SECRET is a long random string (min 64 chars) — generate: openssl rand -base64 64
  • SMTP credentials set to real Gmail App Password or transactional email service
  • Google OAuth Client ID/Secret set from Google Cloud Console
  • Chapa live key set (not test key)
  • Chapa webhook secret matches dashboard configuration
  • FRONTEND_URL set to actual frontend domain
  • HTTPS enabled (SSL certificate installed)
  • uploads/ directory writable: chmod -R 755 uploads/
  • Firewall configured — only ports 22, 80, 443 open: ufw allow 22 80 443 && ufw enable
  • PM2 startup configured: pm2 startup
  • Log rotation enabled: pm2 install pm2-logrotate
